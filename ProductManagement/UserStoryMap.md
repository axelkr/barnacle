# User Story map
## Users (in order of priority)
1. Team
   - responsible for a swimlane
   - objectives: Create value & continuously improve
2. Individual team member
   - Part of a team
   - Responsible for specific tasks / parts of the swimlane
   - Where are the problems in my area of influence / responsibility
3. Team of teams (or any other larger organizational unit)
   - Responsible for several swimlanes
   - How are we doing across swimlanes?
   - Where to focus for improvement?

## Goals (in order of priority)
1. How are we doing? Which part of the swimlane should we look into for improving?
2. Plug into as many different project management software as possible.
3. Explore non-linear state models

## No goals
1. Be a better project management software than those out there

## Red thread
### Setup software
- Prepare environment so teams can plug into project management software
1. Provide Backend, Webpage, application within organization
2. Select backend to connect to in webpage / application
   - Connect to a specific backend 
      - stored across restarts of UI
3. Select parts to focus on
   - UI persists selection of parts to focus on
   - Select states of state model to focus on
      - initial selection: all states but the initial and the done states
   - Select projects to focus on
      - initial selection: all projects
